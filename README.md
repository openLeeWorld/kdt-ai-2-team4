# 피싱/스미싱 문자 판별 프로젝트

## 1. 프로젝트 개요

본 프로젝트는 문자 메시지 형태의 짧은 텍스트 데이터를 분석하여 일반 메시지, 스팸 메시지, 피싱 메시지 등을 분류하는 인공지능 기반 텍스트 분류 프로젝트입니다.

최근 피싱 문자, 스미싱, 광고성 메시지, 악성 링크가 포함된 메시지가 증가하면서 사용자가 위험한 메시지를 사전에 인지할 수 있는 시스템의 필요성이 커지고 있습니다. 본 프로젝트는 이러한 문제를 해결하기 위해 텍스트 분류 모델을 활용하여 메시지의 위험 여부를 자동으로 판단하는 것을 목표로 합니다.

본 프로젝트는 부트캠프 팀 프로젝트의 일환으로 진행되었습니다.

---

## 2. 프로젝트 목표

본 프로젝트의 주요 목표는 다음과 같습니다.

- 문자 기반 텍스트 데이터 수집
- 피싱, 스팸, 일반 메시지 분류를 위한 데이터 전처리
- 머신러닝 및 딥러닝 기반 분류 모델 학습
- 모델을 기반으로 메시지 분류를 하는 플랫폼 제작

## 6. 데이터셋

본 프로젝트는 메시지 본문과 라벨이 포함된 텍스트 데이터셋을 사용합니다.

예시 데이터 형식은 다음과 같습니다.

| text                                                      | label    |
| --------------------------------------------------------- | -------- |
| 배송 주소 확인이 필요합니다. 아래 링크를 눌러 확인하세요. | phishing |
| 오늘 오후 3시에 회의가 있습니다.                          | normal   |

## 7. 실행 방법

### 웹 MVP

```bash
cd frontend/web_mvp
npm install
npm run dev
```

기본 API 엔드포인트는 `/predict`이며, 다른 백엔드 주소를 사용할 경우 `frontend/web_mvp/.env.local`에 `VITE_SMISHING_API_URL`을 지정합니다.

## 10. 한계점

- 공개된 한국어 피싱 및 스팸 문자 데이터셋이 제한적입니다.
- 데이터 품질과 라벨 정확도에 따라 모델 성능이 크게 달라질 수 있습니다.
- 클래스 불균형이 존재할 경우 특정 클래스에 대한 예측 성능이 낮아질 수 있습니다.
- 새로운 유형의 피싱 문자는 기존 학습 데이터만으로 탐지하기 어려울 수 있습니다.
- 실제 서비스 적용 시 개인정보 보호, 보안, 법적 검토가 필요합니다.
