# 프로젝트 기여 가이드

우리 프로젝트에 기여해 주셔서 감사합니다! 원활한 코드 반영을 위해 아래 PR 절차를 지켜주세요.

## 🚀 Pull Request 절차

1. **Issue 확인**: 수정하려는 내용이 Issue에 등록되어 있는지 확인하고, 없다면 새로 생성합니다.
2. **Fork & Clone**: 본 저장소를 Fork한 후 로컬 환경에 Clone 합니다.
3. **Branch 생성**: 작업할 기능에 맞는 브랜치를 생성합니다.
   - 예: `feature/login-fix` 또는 `bugfix/issue-123`
4. **Commit 컨벤션**: 변경 사항을 커밋할 때는 명확한 메시지를 작성하세요.
   - 예: `Feat: 구글 로그인 기능 추가`
5. **PR 제출**: 작업이 완료되면 `main` 브랜치를 대상으로 PR을 보냅니다.
6. **리뷰 및 병합**:
   - 모든 PR은 최소 **3명 이상의 담당자 승인(Approve)**을 얻어야 `main` 브랜치로 병합될 수 있습니다.
   - `main` 브랜치 병합이 완료되면, 원격(Remote)에 생성했던 본인의 작업 브랜치는 관리를 위해 **즉시 삭제**해 주세요.

## ✅ PR 체크리스트

- [ ] 관련 Issue 번호를 PR 설명에 기재했나요? (예: Closes #1)
- [ ] 기존 테스트 코드를 통과했나요?
- [ ] 코드 스타일 가이드를 준수했나요?

## 💬 PR 리뷰 응답

리뷰어가 수정을 요청하면 해당 브랜치에서 추가 커밋을 해주세요. 승인(Approve) 후 메인 코드에 병합됩니다.
